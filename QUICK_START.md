# Quick Start Guide

Get your ChatGPT App running in 5 minutes!

## Prerequisites

- Node.js 18+ installed
- ngrok account (free tier works)
- ChatGPT Plus or Team account

## Setup Steps

### 1. Install Dependencies

```bash
npm run install-all
```

This installs dependencies for both the server and web component.

### 2. Build the Component

```bash
npm run build
```

This compiles the React component into a bundle that the server can serve.

### 3. Start Development Environment

**Terminal 1 - Start the server:**
```bash
npm run dev
```

This starts the MCP server on port 2091.

**Terminal 2 - Start ngrok tunnel:**
```bash
npm run setup-ngrok
```

This:
- Configures ngrok with your auth token (from `.env`)
- Starts an HTTPS tunnel to your local server
- Displays your public HTTPS URL

### 4. Connect to ChatGPT

1. Copy the HTTPS URL from ngrok (e.g., `https://abc123.ngrok-free.app`)
2. Go to [ChatGPT](https://chatgpt.com)
3. Click Settings (bottom left) → Apps → Developer
4. Add your app:
   - **URL**: `https://your-ngrok-url.ngrok-free.app/mcp`
   - Click "Add"

### 5. Test Your App

In ChatGPT, try:
```
Say hello to Claude
```

The app should invoke the `hello_world` tool and display a greeting card!

## What Just Happened?

1. **MCP Server** (`server/server.js`) is running locally on port 2091
2. **ngrok** creates an HTTPS tunnel from the internet to your local server
3. **ChatGPT** connects to your server via ngrok
4. When you mention "hello" in chat, ChatGPT discovers and invokes your tool
5. The server returns both data (for the model) and a React component
6. ChatGPT renders your component in an iframe

## Development Workflow

### Making Changes to the Component

1. Edit `web/src/component.tsx`
2. Rebuild: `npm run build`
3. Restart server: `Ctrl+C` in Terminal 1, then `npm run dev`
4. Test in ChatGPT (may need to refresh the page)

### Making Changes to the Server

1. Edit `server/server.js`
2. Restart server: `Ctrl+C` in Terminal 1, then `npm run dev`
3. Test in ChatGPT

### Watching for Component Changes

For faster development, run this in a third terminal:

```bash
cd web
npm run watch
```

This automatically rebuilds the component when you save changes.

## Troubleshooting

### "Component bundle not found"
**Solution**: Run `npm run build` before starting the server.

### "Port 2091 already in use"
**Solution**:
```bash
lsof -ti:2091 | xargs kill
```
Or change `PORT` in `.env` to a different number.

### "ngrok auth token not set"
**Solution**: Make sure you have a `.env` file with `NGROK_AUTH_TOKEN=your_token`

### Component not showing in ChatGPT
**Solutions**:
1. Verify ngrok URL is correct in ChatGPT settings
2. Check that `/mcp` is appended to the ngrok URL
3. Refresh ChatGPT page
4. Check server logs for errors
5. Verify component builds successfully

### Changes not appearing
**Solution**:
1. Rebuild component: `npm run build`
2. Restart server
3. Hard refresh ChatGPT (Cmd+Shift+R)

## Security Note

⚠️ **Important**: Your ngrok auth token is stored in `.env` which is gitignored. Never commit `.env` to version control. If you accidentally expose your token, regenerate it at [ngrok.com/dashboard](https://dashboard.ngrok.com/get-started/your-authtoken).
